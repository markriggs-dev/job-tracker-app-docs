Job Tracker: Architecture Overview

The Job Tracker is a microservices system built around deliberate architectural tradeoffs. The Job Service publishes to Kafka and returns 202 immediately — the consumer handles the DB write asynchronously. All traffic enters through a YARP gateway on port 5000, which validates Auth0 JWTs before routing downstream. SignalR at the consumer layer pushes real-time updates back to the React frontend via TanStack Query invalidation.
